# Product

## Register

product

## Platform

web

## Users

Maniflight is for maintainers and small engineering teams preparing a GitHub repository for collaboration, release, or production use. They need a fast, evidence-backed view of repository readiness without handing source code or secrets to a hosted service.

## Product Purpose

Maniflight performs read-only diagnostics across repository architecture, automation, security hygiene, community readiness, and live pull-request state. It produces transparent JSON plus accessible human output that makes blockers and evidence gaps easy to inspect and act on. Success means a maintainer or contributor can understand what was checked, what was not knowable, and the next concrete action without mistaking the result for a certification.

## Positioning

GitHub diagnostics that show their evidence.

## Brand Personality

Precise, exploratory, and calm. Maniflight borrows the language of mission control and orbital systems while behaving like serious engineering tooling: direct labels, restrained motion, and no decorative noise between a maintainer and the evidence.

## Anti-references

Maniflight must not resemble a badge wall, a gamified score toy, a generic purple SaaS dashboard, a security-certification seal, or a dense architecture diagram that values spectacle over comprehension. It avoids unreliable hosted widgets, fake telemetry, excessive glass effects, constant ambient animation, and hover-only interactions.

## Design Principles

1. Evidence before score: every judgment links to a documented rule and concrete repository evidence.
2. Unknown is honest: inaccessible or missing context lowers confidence rather than becoming a silent pass or fail.
3. Read-only by default: scanning never executes repository code and never mutates the inspected project.
4. Motion explains state: orbital movement appears only around active focus and disappears for reduced-motion users.
5. One report, every viewport: the text summary remains primary, while the visual system adds fast navigation on desktop and collapses cleanly on mobile.

## Accessibility & Inclusion

The standalone report targets WCAG 2.2 AA. It supports keyboard navigation, visible focus, semantic landmarks, non-color status labels, 200% zoom, a 320px-wide viewport, light and dark color schemes, print, and `prefers-reduced-motion`.
